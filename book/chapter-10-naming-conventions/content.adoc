= Naming Conventions

== Files

* Chapters: `NN-topic.adoc`
* Figures: descriptive names like `architecture.svg`
* Code samples: descriptive names like `gpio_blink.c`
* Keep names stable once published

== Chapters

* Use short, action-oriented titles
* Prefer one topic per chapter
* Keep numbering fixed for Git history

== Code

* One file per concept when possible
* Use lowercase with underscores
* Match example filenames in comments and prose

== Figures

* Prefer SVG for diagrams
* Use names that describe the content, not the tool
* Keep source files if a diagram is exported
